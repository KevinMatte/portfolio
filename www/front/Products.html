Distributed Computing Solutions

Safe Client Callbacks

A seamless solution for Server/Client applications (especially web based) with Client Remote Objects in a secure network.

Allows Clients to sit behind firewalls or sit within unreachable subnets and still have full Server/Client capabilities. Only the Server must be accessible to the Client.

Platform

Server/Client Applications using JAVA RMI

Background

In a Server/Client application, a server may be placed in a secure location behind a firewall with specific ports open for communication.

On the Server, at least two ports must be reachable for a Client to make contact to it. They are:

  • The RMIRegistry's port, where the library of Server Remote Objects are stored.
  • One port for each remote object on the Server.

Without this product, for every Remote Object on a client, ports must be controlled allowed through firewalls and routed through networks.

With this product, no client ports are opened, and yet, the server can communicate with Remote Objects on the Client through a new Client/Server protocol.
